# API 集成（API Integration）

并非每个服务都有一流的集成——但你的应用可以直接连接到**任何外部 API**。通过使用标准 HTTP 请求，你可以与几乎任何平台集成、获取或发送数据，并触发工作流，而无需预构建的连接器。

这为你提供了使用现代 REST、GraphQL 或其他 Web API 的完全灵活性。

## 你可以做什么

通过自定义 API 调用，你的应用可以：
- 连接到 Web 上的**任何 REST 或 GraphQL API**。  
- 从外部服务**发送和接收数据**。  
- 触发操作，如创建记录、发送消息或获取分析数据。  
- 围绕 API 构建**自定义自动化**和工作流。  
- 将 API 调用与其他集成或 AI 操作链接起来，实现强大的流程。

## 步骤 1：获取 API 访问权限

1. 确定你想要连接的服务。  
2. 查看其开发者文档以了解 API 访问要求。  
3. 获取必要的凭据（例如 **API 密钥**、**令牌**或 **OAuth**）。  
4. 使用环境变量安全地存储凭据——**切勿**硬编码密钥。

   *示例：*  
   - **API 密钥：** `sk-xxxxxxxxxxxxxxxx`  
   - **基础 URL：** `https://api.example.com/v1/`

## 步骤 2：与你的应用连接

1. 在 AI Builder 中导航到 `secrets`（密钥）选项卡，并将你的 API 凭据添加为密钥。
2. 然后提示 AI 使用这些密钥来做你想做的事情，它将安装必要的库并为你设置 API 调用。

## 步骤 3：注意事项

* **保护密钥安全：** 使用环境变量或密钥存储来存放 API 密钥。
* **检查速率限制：** 许多 API 有请求限制——请相应地构建。
* **与 AI 或其他集成结合：** 例如，通过 API 获取数据并使用 LLM 进行摘要。


通过 API 集成，你可以将应用连接到**几乎任何现代平台或服务**，为你提供超越原生集成的无限可扩展性。
