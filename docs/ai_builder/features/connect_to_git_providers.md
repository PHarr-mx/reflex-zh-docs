# 连接到 Git 提供商

除了 [GitHub 集成](/docs/ai/features/connect-to-github/)之外，Reflex Build 还可以同步到任何 Git 远程仓库，包括 GitLab、Bitbucket、自托管服务器或现有的 GitHub 仓库。你使用仓库 URL 和细粒度访问令牌进行连接，然后以与 GitHub 相同的方式进行推送和拉取。

## 你需要什么

要连接，请提供：

- **远程 URL**：仓库的 HTTPS 克隆 URL（例如 `https://gitlab.com/your-org/your-app.git`）。
- **分支**：要同步的分支。指向现有分支，或为空仓库命名一个新分支。
- **细粒度访问令牌**：来自你的提供商的具有读写仓库权限的令牌。它用于认证推送和拉取。

与 GitHub 集成（每个用户使用自己的账户授权）不同，该令牌作为应用的密钥之一（`GIT_PAT_TOKEN`）保存。任何在应用上工作的人都使用该存储的令牌进行推送和拉取，因此请使用其仓库访问权限适合与应用协作者共享的令牌。

## 工作原理

该连接跟踪两个独立的仓库：Reflex Build 为你的应用保存的内部历史记录，以及你的外部远程仓库。推送会将应用的当前文件作为提交复制到远程仓库；拉取会将远程仓库的最新文件复制回编辑器。两个历史记录保持独立，因此同步会跨仓库复制文件，而不是单个提交。

## 连接

1. 在编辑器中打开 Git 连接对话框，选择通用 Git 选项。
2. 输入远程 URL、分支和你的细粒度访问令牌。
3. 选择是否在连接后立即将远程仓库的文件拉取到编辑器中。

连接后，**Push**（推送）和 **Pull**（拉取）操作与 GitHub 相同。

## 连接到现有仓库

要引入现有仓库，输入其远程 URL 和具有访问权限的令牌，然后将其文件拉取到编辑器中。此后 Reflex Build 将针对该分支进行同步。

如果仓库为空，请命名一个新分支而不是现有分支，Reflex Build 将在首次推送时初始化它。

## 要求

Git 集成在包含 Git 连接功能的计划中可用。如果你的计划不包含它，连接时将提示你升级。
