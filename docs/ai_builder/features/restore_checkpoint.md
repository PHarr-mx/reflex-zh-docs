# 恢复检查点（Restore Checkpoint）

**恢复检查点**功能允许你将应用回滚到 AI Builder 对话期间的任何先前状态。当你想撤消最近的更改并返回到应用的较早版本时，这非常有用。

## 工作原理

每次 AI 代理对你的应用进行更改时，都会自动创建一个检查点。你可以随时恢复到这些检查点中的任何一个，有效地撤消该点之后所做的所有更改。

## 使用恢复检查点

1. **找到恢复图标**：在每条对你的应用进行更改的 AI 代理消息的末尾，你会看到一个圆形箭头图标（↻）。

2. **点击恢复**：点击你想恢复到的消息旁边的圆形箭头图标。

3. **确认操作**：应用将恢复到该特定消息处理后的确切状态。

4. **继续构建**：恢复后，你可以继续对话并从该点进行新的更改。

## 何时使用恢复检查点

- **撤消不需要的更改**：当 AI 做出了你不喜欢的更改时
- **尝试不同的方法**：恢复并要求 AI 以不同的方式实现功能
- **修复损坏的功能**：当新更改破坏现有功能时回滚
- **安全实验**：测试不同的解决方案，知道你可以随时恢复到检查点

## 重要说明

- 恢复将**永久删除**所选消息之后所做的所有更改
- 你无法撤消恢复操作——请谨慎选择你的恢复点
- 对话历史保持完整，但恢复点之后的代码更改将丢失
- 恢复检查点仅影响你当前的构建会话

> **提示：** 在进行重大更改之前，请注意哪条消息代表你最后一个稳定的检查点，以便在需要时轻松恢复。
