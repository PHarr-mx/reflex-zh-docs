# Webhooks

Webhooks 允许你的应用实时**向外部服务发送数据**。你向 AI Builder 提供在另一个平台中创建的 **webhook URL**，当工作流被触发时，它可以**自动向该 URL 发送有效载荷**。

这是一种简单而强大的方式，可以与支持传入 webhook 的服务集成，即使没有一流的集成。

## 你可以做什么

通过传出 webhook，你的应用可以：

* 向任何支持传入 webhook 的服务（例如 Slack、Zapier、Make、Discord）**发送结构化有效载荷**。
* 当应用中发生事件时**触发外部工作流**。
* 无需编写自定义后端代码即可**向第三方系统推送实时更新**。
* 将 webhook 调用与其他集成或 AI 操作链接起来。

## 步骤 1：在外部服务中创建 Webhook

1. 转到你想要连接的外部服务（例如 Slack、Zapier、Discord、Make）。
2. 在该服务中创建一个新的**传入 webhook**。
3. 复制它提供的 **webhook URL**。


## 步骤 2：将 Webhook URL 添加到 AI Builder

1. 在 AI Builder 聊天中粘贴你在外部服务中创建的 **webhook URL**。
2. 然后你可以指示 AI 在工作流被触发时**向此 URL 发送数据**。
3. 可选地定义**有效载荷结构**（例如 JSON 正文）以及何时应该发送。

> 💡 示例："每当用户注册时，将用户注册数据发送到此 webhook。"

## 步骤 3：发送有效载荷

* AI Builder 将编写代码以向你定义的有效载荷的 webhook URL 发送 `POST` 请求。
* 有效载荷可以包含来自你应用的**动态数据**——例如用户信息、状态变量或模型输出。
* 你可以从按钮、事件、工作流或自动化触发这些 webhook 调用。


## 步骤 4：注意事项

* **安全性：** 仅使用来自受信任服务的 webhook URL——它们将按原样接收你的数据。
* **格式：** 确保有效载荷与外部服务预期的格式匹配。
* **链接：** 你可以使用多个 webhook 或将它们与其他集成结合使用。
* **用例：** Slack 警报、CRM 更新、在 Zapier 或 Make 中触发自动化、通知自定义系统。
